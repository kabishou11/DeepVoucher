# 自动录入凭证系统实施计划

日期：2026-03-19

## 1. 实施目标

在本地环境中交付一套可运行的自动录入凭证系统，支持：

- 上传一组图片
- 对图片进行多模态理解
- 生成 1 张支持多借多贷的凭证草案
- 在可视化界面中展示完整处理过程
- 在存在歧义时进入人工确认
- 最终导出符合接口格式的 JSON

## 4. 推荐实施顺序

1. 先搭工程和配置，不先接真实模型
2. 先打通常量输入到静态假数据流程
3. 再接入知识层与规则层
4. 再接入 ModelScope 多模态抽取
5. 再做前端可视化与人工确认
6. 最后做端到端验收和回归

这样可以避免一开始把复杂度堆在同一阶段。

## 5. 风险与控制

### 风险 1：图片事实抽取不稳定

控制：

- 模型输出强制落 schema
- 增加标准化层
- 冲突进入阻断，而不是硬提交

### 风险 2：科目匹配到父级节点

控制：

- 科目树解析后只允许末级入账
- 父级节点只参与导航与解释

### 风险 3：多借多贷拆分失真

控制：

- 拆分仅基于证据可支撑的金额单元
- 金额闭合不成立时直接阻断

### 风险 4：答案泄漏

控制：

- `正确答案` 只进入测试断言
- 推理代码层显式禁用 `ground_truth` 目录
