## 已完成能力

- 一组附件默认生成一张凭证，支持多借多贷
- 科目匹配遵循科目层级与末级科目约束
- 借贷不平衡、日期缺失、科目缺失、金额残差等情况会阻断导出
- 真实多模态调用失败时不再回退 mock 放行，而是直接阻断
- 当前样本可稳定落到目标三行分录和月末凭证日期
- 系统开始具备“越用越聪明”能力：只从已确认/已导出的结果中积累摘要->科目经验

## 当前仍未完全收尾

- LanceDB 在当前受限沙箱中仍可能因底层文件重命名失败而退回 JSON fallback，但在放权环境已验证可用
- LangGraph 仍是规划层概念，主干仍为函数式 pipeline
- 多借多贷能力已可用，但通用规则库还不够厚，当前对真实复杂样本覆盖仍不足
- 真实 live 回归仍需要继续扩大样本集，不应只以当前样本作为泛化依据

## 风险

- 最大风险是当前样本已稳定，但通用样本仍可能出现新的拆分歧义
- 文档口径与实现口径仍有少量差距，尤其是 LangGraph / embedding / LanceDB 真正启用程度
- “100% 准确率”当前依赖强校验和人工阻断闭环，而不是任意票据都能自动直出
